# Tech / Semiconductor ETF Analyses

> **TL;DR**: Every **cap-weighted** tech/semi ETF re-imports the NVDA concentration this book's
> core deliberately avoids → **all of them fail the
> [[foundation/methodology/execution-vehicle-typology|role-(a)]] core wrap-quality gate.** There is
> no cap-weighted fund that serves as clean role-(a) core tech exposure for an NVDA-excluding book.
> The role-(c) NVDA-diluter category — VVSM (10%-capped), XSD (equal-weight), or IGV
> (software-only, 0% semis) — is a structurally different construction, used as a satellite overlay
> rather than a core holding. The durable content is the role + NVDA-weight construction analysis;
> weights are dated snapshots. *Analysis, not advice.*

**Construction rule**: role-(a) core exposure for this book = none of the cap-weighted funds (each
re-imports NVDA). The role-(c) diluters function as satellite overlays layered on top of the
direct-name core, not as a core substitute. **Verify VVSM's 10%-cap + top holdings on the live
factsheet at processing** (AMD 10.3% / AVGO 9.6% / MU 9.4% / TSM 8.8% / NVDA 8.4% as collected).

## Why this is pinned (don't re-litigate)

The sweep confirmed hard that cap-weighting *is* NVDA-concentration — documenting it once prevents
re-deriving it each session and pins VVSM as the verified role-(c) overlay. The wrap-quality gate
(the "right names, wrong weights" failure mode in the
[[foundation/methodology/execution-vehicle-typology|typology]]) is exactly what fails these as
cores.
